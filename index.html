Why Javascript?

Javascript is the most popular lanugage used on the web today and has changed a lot since I started to code in late 2013.

I began web development as a web designer and was first introduced to Javascript through, a popular Javascript library, jQuery. I used jQuery sparingly to add animations and interactivity to my web pages. As interactive web pages became more popular, I started to use it more often on client pages. With different clients came different problems to solve with jQuery.

I realized I enjoyed the problem solving aspect of coding more than the designing of web pages and dove straight into jQuery documentation (which I realize now may have not been the best approach to learning a coding language or learning coding in general). I learned, at best, to imitate good jQuery code and to get code to work with little to few work-arounds but never understood how the library worked and how it differed from regular Javascript code.

But as a self-taught web developer I continued to "learn" this way and soon discovered frameworks like Angular which led to an interest in web app development. An interest in web app development naturally led to an interest in backend development (Node) and I soon realized I needed to learn fundamental Javascript before I could write good and effiecient code as a full stack developer.

Thus bringing us to why I'm starting this blog and an answer to the title of this first post, 'Why Javascript?'. Simply put, I've been coding in Javascript for 6 years and I have never learned the fundamentals of Javascript, best practices or grasped a true understanding of the language. So through this blog I plan to master this langauge, and all its oddities, and finally code confidently in Javascript.
